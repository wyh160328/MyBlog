一、MyBatis简介

  1. Mybatis开源免费框架，原名叫做iBatis,2010年在google code，2013年迁移到GitHub

  2. 作用

    2.1底层是对JDBC的封装

  3. Mybatis优点之一：

    3.1使用mybatis时不需要编写实现类，只需要写需要执行的sql命令

二、环境搭建

  1. 导入jar

​ asm–cglib依赖的包

​ cglib–动态代理的包

​ commons-logging–日志包

​ javassist–字节码解析包，也是cglib依赖的包

​ mybatis–框架核心包

​ mysql-connector–驱动包

​ 其他log字样的包都是日志所需的jar包

  1. 在src下新建全局配置文件（编写JDBC四个变量）

​ 2.1没有名称和地址要求

​ 2.2在全局配置文件中引入DTD或者schema(引入的代码可从MyBatis的帮助文档中获取)

​ 2.2.1如果导入dtd后没有提示

​ Window–>preference–>XML–>XMI catalog–>add按钮

​ 选择本地下载好的dtd文件

​ 2.3全局配置文件内容

2.4环境搭建详解

​ 2.4.1.全局配置文件中内容

​ type 属性可取值

​ 1 JDBC,事务管理使用 JDBC 原生事务管理方式

​ 2 MANAGED 把事务管理转交给其他容器.原生 JDBC 事务setAutoMapping(false);

​ type 属性

​ 1 POOLED 使用数据库连接池

​ 2 UNPOOLED 不使用数据库连接池,和直接使用 JDBC 一样

​ 3 JNDI :java 命名目录接口技术.

三、三种查询方式

  1. selectList() 返回值为 List<resultType 属性控制>

​ 适用于查询结果都需要遍历的需求

  1. selectOne() 返回值 Object,

​ 适用于返回结果只是变量或一行数据时

  1. selectMap() 返回值 Map

​ 3.1 适用于需要在查询结果中通过某列的值取到这行数据的需求.

​ 3.2 Map<key,resultType 控制>

四、MyBatis的工作原理

  1. 运行过程中涉及到的类
    1.1 Resources MyBatis 中 IO 流的工具类——加载配置文件
1.2 SqlSessionFactoryBuilder() 构建器
    1.2.1 作用:创建 SqlSessionFactory 接口的实现类
1.3 XMLConfigBuilder MyBatis 全局配置文件内容构建器类
    1.3.1 作用负责读取流内容并转换为 JAVA 代码.
1.4 Configuration 封装了全局配置文件所有配置信息.
    1.4.1 全局配置文件内容存放在 Configuration 中
1.5 DefaultSqlSessionFactory 是SqlSessionFactory接口的实现类
1.6 Transaction 事务类
    1.6.1 每一个 SqlSession 会带有一个 Transaction 对象.
1.7 TransactionFactory 事务工厂
    1.7.1 负责生产 Transaction
1.8 Executor MyBatis 执行器
    1.8.1 作用:负责执行 SQL 命令
    1.8.2 相当于 JDBC 中 statement 对象(或 PreparedStatement或 CallableStatement)
    1.8.3 默认的执行器 SimpleExcutor
    1.8.4 批量操作 BatchExcutor
    1.8.5 通过 openSession(参数控制)
1.9 DefaultSqlSession 是 SqlSession 接口的实现类
1.10 ExceptionFactory MyBatis 中异常工厂
