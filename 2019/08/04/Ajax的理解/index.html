Ajax的理解

  • 2019-08-04

一、什么是Ajax

​ AJAX全称为“Asynchronous JavaScript and XML”（异步JavaScript和XML），是一种创建交互式网页应用的网页开发技术。

​ 不是一种新技术，是如下几种技术的组合应用：

​ *基于web标准（standards-based presentation）XHTML+CSS的表示；

​ *使用 DOM（Document Object Model）进行动态显示及交互；

​ *使用 XML 和 XSLT 进行数据交换及相关操作；

​ *使用 XMLHttpRequest 进行异步数据查询、检索；

​ *使用 JavaScript 将所有的东西绑定在一起。

Ajax本质上是一个浏览器端的技术。

Ajax的优点

​ 1.通过异步模式，提升了用户体验

​ 2.优化了浏览器和服务器之间的传输，减少不必要的数据往返，减少了带宽占用

三、传输数据的格式说明

  1. 字符串格式

    ​ 后端可以直接通过HttpServletResponse实例中的getWriter().write(）方法将我们需要的数据传输给前端发送请求的ajax对象。然后前端再根据自身需求对字符串进行截取，拆分等操作。

  2. json格式（最推荐）

    ​ 在后端获取完数据后，数据量往往都非常庞大，如果按照普通字符串的格式发送给前端，这样字符串处理起来的效率严重下降，因此，经常在开发中使用Gson().toJson()方法将我们获取到的对象或者集合转成json对象格式的字符串。前端接收到字符串后。可以直接使用eval将其转会json对象。这样在使用数据时就是以一种操作对象的形式在使用了（数据和对象转换都是实时的，因此维护起来方便）

  3. XML格式

    发送数据时可以采用字符串拼接的方式写成xml格式的字符串。前端使用ajax.responseXML的方式去接受传送过来的对象。此时便可以想dom操作一样对数据进行使用。

    缺点:数据量大时，维护起来麻烦。特别不推荐

四、open()方法的参数详解

  1. method:有get和post两个取值，数据请求的两个方式

    若是get方式请求数据，提交的信息写在”url?”后面，并且以键值对的形式存在，多对则用“&”隔开，send函数发送null

    若是用post方式请求数据，提交的信息由send函数发送。并且要在open（）函数底下设置请求头

  1. url:请求数据的url地址

  2. boolean:true/false，选择异步和同步（默认异步）

    当该boolean值为true时，服务器请求是异步进行的，也就是脚本执行send（）方法后，不等待服务器的执行结果，而是继续执行脚本代码；

    当该boolean值为false时，服务器请求是同步进行的，也就是脚本执行send（）方法后，等待服务器的执行结果的返回，若在等待过程中超时，则不再等待，继续执行后面的脚本代码！
